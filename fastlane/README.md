# App Store metadata automation

This directory automates the **text** fields on the App Store Connect product
page — app name, subtitle, promotional text, description, keywords, and the
support / marketing / privacy-policy URLs — with
[fastlane deliver](https://docs.fastlane.tools/actions/deliver/).

It does **not** upload the app binary or screenshots. The binary is still built
and uploaded locally through Xcode (see [`../RELEASE_IOS.md`](../RELEASE_IOS.md)).
Metadata and the binary are two separate things in App Store Connect; this
handles only the former.

## Layout

```
fastlane/
  Appfile                     app identifier
  Deliverfile                 deliver config (text-only, API-key auth)
  asc_api_key.example.json    template for your API key (copy → asc_api_key.json)
  metadata/
    copyright.txt             app-level fields (edit freely)
    primary_category.txt      FINANCE
    secondary_category.txt    PRODUCTIVITY
    <locale>/                 one folder per shipped language (16 total)
      name.txt  subtitle.txt  promotional_text.txt
      description.txt  keywords.txt  release_notes.txt
      support_url.txt  marketing_url.txt  privacy_url.txt
    review_information/       optional; needs first_name, last_name,
                              email_address, phone_number (+notes) or
                              deliver fails — park incomplete notes under
                              review_information.local/ instead
```

The tree is generated by [`../scripts/gen-appstore-metadata.mjs`](../scripts/gen-appstore-metadata.mjs).
Every locale under `metadata/` holds **final native copy** (localized display
names included) — see [`TRANSLATION_STATUS.md`](TRANSLATION_STATUS.md).

The URL fields point at the landing site ([`../website/`](../website/README.md),
deployed to GitHub Pages): `…/support.html`, the site root as marketing URL,
and `…/privacy.html`. They must be **public** HTTPS pages (Apple precheck GETs
them), so make sure the site is deployed before pushing metadata. If the
hosting ever moves, change `WEBSITE_BASE` in
[`../scripts/gen-appstore-metadata.mjs`](../scripts/gen-appstore-metadata.mjs)
and regenerate the three `*_url.txt` files in every locale.

## One-time setup

1. **Install fastlane:** `brew install fastlane` (or `gem install fastlane`).
2. **Create an App Store Connect API key:** App Store Connect → Users and Access
   → Integrations → App Store Connect API → generate a key with the *App Manager*
   role. Download the `AuthKey_XXXX.p8` (you can only download it once) and note
   the **Key ID** and **Issuer ID**.
3. **Wire the key in:** copy the template and fill it in —
   ```bash
   cp fastlane/asc_api_key.example.json fastlane/asc_api_key.json
   ```
   Set `key_id`, `issuer_id`, and paste the `.p8` PEM body into the `key`
   field (deliver reads the key inline from this JSON). `asc_api_key.json` is
   git-ignored.
4. **Deploy the landing site** (once): repo Settings → Pages → Source:
   GitHub Actions. The support / marketing / privacy URLs in the metadata
   point at it and must resolve publicly before App Review.

## Pushing metadata

```bash
npm run release:ios:metadata      # runs guards, then `fastlane deliver`
```

By default deliver shows an HTML preview and waits for confirmation. For an
unattended push (CI), set `FORCE=1`.

The command aborts early if fastlane is missing, the API key file is missing, or
any `REPLACE_ME` placeholder remains in the metadata.

## Editing copy

Edit the `.txt` files directly — they are the source of truth and are committed.
Re-running the generator never overwrites an existing file (pass `--force` to
regenerate from the built-in copy). Apple length limits: name/subtitle 30,
promotional text 170, keywords 100, description 4000; the generator validates
the first three and fails if any are exceeded.

## Not handled here

Screenshots, app preview videos, pricing/in-app purchases, and the App Privacy
questionnaire are still managed in the App Store Connect UI. (The privacy
policy **URL** is pushed from `metadata/<locale>/privacy_url.txt`; the
questionnaire answers are not.)
