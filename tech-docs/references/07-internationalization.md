# 07 — Internationalization (i18n)

The app ships **16 languages + a Device (follow-system) option**, with device-locale detection, a
per-locale glossary, and a CI parity test that keeps every locale key-for-key in sync with English.

## Supported languages

`en` English · `zh` Traditional Chinese (繁體) · `es` Spanish · `fr` French · `de` German · `ja` Japanese ·
`hi` Hindi · `pt` Portuguese · `ru` Russian · `id` Indonesian · `ko` Korean · `it` Italian · `tr` Turkish ·
`vi` Vietnamese · `th` Thai · `pl` Polish — plus **`device`** (follow-system).

The canonical list lives in three places that must agree: the `Language` type
(`contexts/LanguageContext.tsx`), `SUPPORTED_LANGUAGES` (`utils/constants.ts`), and `getLanguageOptions`
(`utils/language-options.ts`). In the picker, the first 6 are translated via `t(...)`; the 10 newer locales
are shown as **endonyms** in their own script (हिन्दी, Русский, 한국어, ไทย, …).

## Where the strings live

- **Main UI dictionary**: a large inlined `translations: Record<ResolvedLanguage, Record<string,string>>` in
  `contexts/LanguageContext.tsx` (~3,770-line file). The English block alone is ~212 keys covering
  accounting, statistics, diagram, auth, cloud sync, storage/IAP plans, and errors. All 16 locales are in
  this one file.
- **Glossary / category strings**: `utils/glossary-i18n.ts` (~722 lines) exports 16 dictionaries
  `GLOSSARY_TRANSLATION_EN … _PL` (~42 keys each — the Smart Glossary UI strings + the 12 `glossary.*`
  category labels), merged into `GLOSSARY_TRANSLATIONS` in `LanguageContext.tsx`.

## Detection, persistence, switching

- **Detection** (`getDeviceLanguage`): reads `expo-localization` `getLocales()`, takes the primary locale,
  prefers `languageCode`, falls back to parsing `languageTag` (`"en-US"`→`"en"`), maps via a `languageMap`
  (includes the legacy `in`→`id` alias), and **defaults to `"en"`** on any miss/error (wrapped in try/catch).
- **Provider** (`LanguageProvider`): state defaults to `"device"`; `resolvedLanguage =
  useMemo(device ? getDeviceLanguage() : language)`.
- **Persistence**: AsyncStorage `@flash_accounting_language`; `loadLanguage()` validates the saved value
  against `SUPPORTED_LANGUAGES`; `setLanguage()` writes then updates state.
- **Lookup** (`t`): `translations[lang][key]` → `GLOSSARY_TRANSLATIONS[lang][key]` → `translations.en[key]`
  → `GLOSSARY_TRANSLATIONS.en[key]` → the raw key. So English is the ultimate fallback and unknown keys pass
  through verbatim.
- **UI**: routes `/select-language` (via `components/LanguageSheet.tsx`) and `/glossary` (via `GlossarySheet`)
  drive switching; consumed everywhere via `useLanguage()`.

## Localized formatting

`utils/formatting.ts` handles locale-aware display: `formatCurrency` (via `toLocaleString`), dates/times with
Today/Yesterday special-casing, and a `LOCALE_MAP` from app language to a BCP-47 tag. Numeric displays use
`fontVariant: ["tabular-nums"]` for aligned digits.

## Glossary localization (see also [06](06-smart-features.md))

- **Trigger terms**: native-language synonyms for the 12 built-in categories exist for **5 languages** (zh,
  ja, es, fr, de) in `utils/glossary-locale-terms.ts`; the other 10 fall back to English terms + the
  translated label.
- **Labels/UI**: all 16 locales fully covered in `utils/glossary-i18n.ts`.

## Parity enforcement — `tests/i18n-parity.test.ts`

Because `LanguageContext.tsx` can't be imported under Node (it pulls in RN modules), the test **parses the
`translations` object out of the source text** via regex, then asserts:

1. Every expected language block exists.
2. The set of language blocks **exactly equals** the 16-locale `EXPECTED_LANGUAGES`.
3. Every locale has **exactly the same key set as English** — no missing keys, **no extra keys**.

A second test applies the identical assertion to the 16 `GLOSSARY_TRANSLATION_*` dictionaries. Net effect:
**adding a language means adding it to `EXPECTED_LANGUAGES` and supplying a complete translation set for both
dictionaries, or CI fails.** This is the mechanism that keeps 16 locales honest.

## i18n reach beyond the app

The same 16-locale spine feeds two external surfaces (kept in sync automatically):

- **App Store metadata** — `fastlane/metadata/<locale>/` has full localized name/subtitle/description/keywords
  for all 16 locales, generated by `scripts/gen-appstore-metadata.mjs`. See [13](13-external-tooling.md).
- **Marketing website** — `website/scripts/sync-content.mjs` reads the same Fastlane metadata so the site and
  the store listing never drift. See [13](13-external-tooling.md).

## RevenueCat paywall locale bridge

RevenueCat's native paywall doesn't read the in-app language picker, so `ProContext` computes a preferred
paywall locale from the app language + device locale and pushes it before presenting (e.g. `zh`→`zh_Hant`,
`pt`→`pt_BR`, with `EXPO_PUBLIC_REVENUECAT_PAYWALL_LOCALE_*` overrides). See [10](10-monetization-and-auth.md).
